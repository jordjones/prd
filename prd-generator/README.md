# prd-generator

A Claude Code skill that registers a `/prd` slash command for generating AI-optimized Product Requirements Documents.

## Quick Start

**Global installation** (available in all Claude Code sessions):

```bash
mkdir -p ~/.claude/skills/prd-generator
cp -r prd-generator/* ~/.claude/skills/prd-generator/
```

**Project-scoped installation** (available only in this project):

```bash
mkdir -p .claude/skills/prd-generator
cp -r prd-generator/* .claude/skills/prd-generator/
```

After installation, type `/prd` in any Claude Code session to begin.

## Usage

```bash
# Start the full intake flow
/prd

# Skip the first question by providing a product description
/prd REST API for task management with due dates and priority levels
```

The skill will guide you through 5-8 adaptive intake questions, then generate a complete PRD with self-audit.

## What It Generates

| File | Purpose |
|------|---------|
| `PRD-<workstream>.md` | Full Product Requirements Document (source of truth) |
| `AGENTS.md` | Codex/OpenAI-compatible context file |
| `.cursor/rules/prd-<workstream>.mdc` | Cursor-compatible rules file |

## How It Works

```
/prd invoked
    │
    ▼
Detect existing PRDs ──→ Update / Replace / New
    │
    ▼
Offer project scan ──→ Auto-detect tech stack
    │
    ▼
Hybrid intake (5-8 questions)
    │  ├─ Detect expertise level
    │  ├─ Adapt question depth
    │  └─ Skip answered questions
    │
    ▼
Generate PRD (EARS format, phased)
    │
    ▼
Self-audit (5 quality gates)
    │  ├─ Testable
    │  ├─ Specific
    │  ├─ Bounded
    │  ├─ Singular
    │  └─ Imperative
    │
    ▼
Refinement loop
    │  ├─ Accept as-is
    │  ├─ Refine sections
    │  ├─ Add a phase
    │  ├─ Regenerate
    │  └─ Audit only
    │
    ▼
Save PRD + AGENTS.md + Cursor rules
```

## Extending

**Customize the template:** Edit `references/PRD-TEMPLATE.md` to change the skeleton structure used during generation.

**Add quality gates:** Add new gate definitions to the self-audit section in `SKILL.md` (Step 6). Each gate needs a name, check description, and flag criteria.

**Modify intake questions:** Edit the hybrid intake flow in `SKILL.md` (Step 4). Add, remove, or reorder questions. Keep the total under 8.

**Add operational rules:** Edit `references/PRD-SPEC-RULES.md` to add numbered rules. The self-audit will automatically check against them.

## Cross-Agent Compatibility

The `/prd` command generates files for multiple AI coding agents:

- **Claude Code** reads the PRD directly as a CLAUDE.md-style spec
- **OpenAI Codex** reads `AGENTS.md` following the open standard format (Overview, Conventions, Commands, Testing, Constraints)
- **Cursor** reads `.cursor/rules/prd-<workstream>.mdc` with frontmatter-based rules

The PRD is always the source of truth. Derived files can be regenerated by running `/prd` again.

## Philosophy

AI coding agents cannot infer intent from omission — everything explicit gets built, everything omitted gets invented or ignored. Traditional PRDs written for humans rely on shared context, follow-up questions, and domain knowledge. AI-optimized PRDs must be self-contained specifications using imperative language (SHALL/WHEN/IF), concrete values, explicit non-goals, and executable verification commands. This skill enforces these principles automatically through its generation rules and self-audit engine.

For the full research framework, see `AI_PRD_FRAMEWORK.md` in the project root.
